Notes on Deep Generative Model

Part I - Autoregressive Model

In this note, I cover some basic aspects of autoregressive generative models, where the learning of data distributions is done in a supervised fashion under the modeling assumption that there exists an ordering of inputs such that the conditional distribution of an input can be compactly represented/parameterized as a function of all inputs that come before it in the ordering. Notable autoregressive architectures include PixelCNN, Transformer, etc.

Generative Model

In generative modeling, we assume that our data set DDD is a set of samples from some probability distribution pdatap_{data}pdata​, and the goal is to be able to approximate such a probability distribution given our access to DDD. One way of doing so is via parametric models. That is, we have a family of probability distributions {pθ:θ∈Θ}\{p_{\mathbf{\theta}}: \mathbf{\theta} \in \Theta \}{pθ​:θ∈Θ} parameterized by some (possibly high dimensional vector) θ\mathbf{\theta}θ, and our goal is to learn a parameter θ∗\mathbf{\theta^*}θ∗ such that

θ∗=arg⁡min⁡θ∈Θd(pθ,pdata), \mathbf{\theta^*} = \arg\min_{\mathbf{\theta} \in \Theta} d(p_\theta, p_{data}),θ∗=argθ∈Θmin​d(pθ​,pdata​),

where d(⋅,⋅)d(\cdot, \cdot)d(⋅,⋅) is some notion of distance between distributions, for e.g. KL divergence. Note that we do not know what pdatap_{data}pdata​ is (that’s why we would like to approximate it), but we have information on pdatap_{data}pdata​ via our training data DDD.

Autoregressive Models

Given a probability distribution ppp and a sample x=(x1,…,xn)\mathbf{x} = (x_1,\dots,x_n)x=(x1​,…,xn​), we can factorize the density as

p(x)=∏i=1np(xi∣x<i),p(\mathbf{x}) = \prod_{i=1}^n p(x_i | \mathbf{x}_{<i}),p(x)=i=1∏n​p(xi​∣x<i​),

where x_<i=(x1,…,xi−1)\mathbf{x}\_{<i} = (x_1,\dots,x_{i-1})x_<i=(x1​,…,xi−1​). Here, we can see where the term autoregressive comes from: given an ordering x1,…,xnx_1,\dots,x_nx1​,…,xn​, the distribution of the xix_ixi​ is dependent on the distribution of all the random variables that come before it: x1,…,xi−1x_1, \dots, x_{i-1}x1​,…,xi−1​.

Now, for a moment, assume that each xix_ixi​ is a Bernouilli random variable, then in order to specify p(x)p(\mathbf{x})p(x), we would need 1=201=2^01=20 parameter for p(x1)p(x_1)p(x1​), 212^121 parameters for p(x2∣x1)p(x_2 \mid x_1)p(x2​∣x1​), 2i−12^{i-1}2i−1 parameters for p(xi∣x_<i)p(x_i \mid \mathbf{x}\_{<i})p(xi​∣x_<i), giving a total of

parameters. This means that the space complexity is exponential in the dimension of the input, rendering the tabular approach unemployable.

Thus, in an autoregressive generative model, one assumes that

  1. If xi∈Rx_i \in \mathbb Rxi​∈R is a continuous distribution, p1p_1p1​ could be a normal distribution with mean and variance specified by fif_ifi​.

Fully-visible sigmoid belief network (FVSBN)

Suppose that xi∈{0,1}x_i \in \{0,1\}xi​∈{0,1} for 1≤i≤n1 \leq i \leq n1≤i≤n. More general input types

The FVSBN and NADE models can be easily extended to other input types such as categorical or continuous, i.e. for example when p1p_1p1​ as discussed above is given by softmax or normal distribution.

For categorical inputs xix_ixi​ with kkk values, we simply let αi∈Rd×k\alpha_i \in \R^{d \times k}αi​∈Rd×k, bi∈Rkb_i \in \R^kbi​∈Rk, and σ\sigmaσ is now the softmax function, where

Autoencoders and MADE for Fast Training

Comparison: Autoregressive models v.s. autoencoders

The autoregressive FVSBN and NADE models seem very similar to an autoencoder. For example, for the NADE, we can think of h=(h1,…,hn)=e(x)h = (h_1,\dots, h_n) = e(\mathbf{x})h=(h1​,…,hn​)=e(x) as the encoded version of x\mathbf{x}x. In the decoding phase, we get x^i=σ(αiThi+bi)=d(h)i=d(e(x))i\hat x_i = \sigma(\alpha_i^T h_i + b_i) = d(h)_i = d(e(\mathbf{x}))_ix^i​=σ(αiT​hi​+bi​)=d(h)i​=d(e(x))i​ as the decoded version. For an autoencoder, we aim to have d(e(x))≈xd(e(\mathbf{x})) \approx \mathbf{x}d(e(x))≈x, and so for binary xix_ixi​’s, the logistic loss is a suitable minimization objective ∑x∈D∑i−xilog⁡x^i−(1−xi)log⁡(1−x^i). \sum_{\mathbf{x} \in D} \sum_i - x_i \log \hat x_i - (1-x_i) \log(1-\hat x_i).x∈D∑​i∑​−xi​logx^i​−(1−xi​)log(1−x^i​).

On the other hand, our objective for generative models like NADE as mentioned in the first section is to find optimal parameter θ∗\theta^*θ∗ that minimizes d(pθ,pdata),d(p_\theta, p_{data}),d(pθ​,pdata​), where ddd is some notion of distance between probability distributions. In the case where ddd is the KL divergence

d(pθ,pdata)=Ex∼pdata[log⁡pdata(x)−log⁡pθ(x)],d(p_\theta, p_{data}) = \mathbb E_{\mathbf{x} \sim p_{data}} [\log p_{data}(x) - \log p_\theta(x)],d(pθ​,pdata​)=Ex∼pdata​​[logpdata​(x)−logpθ​(x)],

minimizing the distance is equivalent to maximizing the log-likelihood

Ex∼pdata[log⁡pθ(x)].\mathbb E_{\mathbf{x} \sim p_{data}} [\log p_\theta(\mathbf{x})].Ex∼pdata​​[logpθ​(x)]. Notice now that since

pθ(x)=∏i=1np(xi∣x<i)=∏i=1nBern(x^i),p_\theta(\mathbf{x}) = \prod_{i=1}^n p(x_i|\mathbf x_{<i}) = \prod_{i=1}^n \mathrm{Bern}(\hat x_i),pθ​(x)=i=1∏n​p(xi​∣x<i​)=i=1∏n​Bern(x^i​),

we get that log⁡pθ(x)=∑ixilog⁡x^i+(1−xi)log⁡(1−x^i). \log p_\theta(\mathbf x) = \sum_i x_i \log \hat x_i + (1-x_i) \log (1- \hat x_i).logpθ​(x)=i∑​xi​logx^i​+(1−xi​)log(1−x^i​).

So, the objective of the generative model to minimize the KL divergence is also maximizing the log-likelihood, which is equivalent to minimize the logistic loss. That is, it looks exactly like the objective of the autoencoder.

However, one distinction between an autoregressive model like NADE and an autoencoder is the probabilistic semantics: in autoregressive models, we have an ordering on the random variables xix_ixi​, and the conditional probability of x^i\hat x_ix^i​ can only depend on x<i\mathbf{x}_{<i}x<i​. On the other hand, there is no such constraint on a vanilla autoencoder.

MADE for fast training

Notice that due to the autoregressive behavior, in order to compute the probability density at an input point x∈Rn\mathbf{x} \in \R^nx∈Rn, due to the different network used to compute p(xi∣x<i)p(x_i|\mathbf{x}_{<i})p(xi​∣x<i​), we need nnn passes in order to compute x^=p(x)=∏i=1np(xi∣x<i)\hat{\mathbf x} = p(\mathbf{x}) = \prod_{i=1}^n p(x_i|\mathbf{x}_{<i})x^=p(x)=∏i=1n​p(xi​∣x<i​). On the other hand, in an autoencoder, we only need a single forward pass in order to compute x^\hat{\mathbf x}x^. The question is how one can make an autoencoder into an autoregressive model, where the probabilistic structure is respected. This is where the Masked Autoencoder for Distribution Estimation (MADE) comes in.

The idea is to prohibit paths in an autoencoder that does not respect the conditional probability structure. For simplicity, suppose n=3n=3n=3 and our ordering is again x1,x2,x3x_1, x_2, x_3x1​,x2​,x3​. Clearly, the conditional probability structure mandates that p(x1)p(x_1)p(x1​) does not depend on the any other inputs while p(x2)p(x_2)p(x2​) can only be dependent on x1x_1x1​, and p(x3)p(x_3)p(x3​) can only depend on x1x_1x1​ and x2x_2x2​. Reasoning backwardly, this also means that paths from x1x_1x1​ can reach x^2\hat x_2x^2​ or x^3\hat x_3x^3​ but not x^1\hat x_1x^1​; paths from x2x_2x2​ can reach x^3\hat x_3x^3​ but not x^1\hat x_1x^1​ or x^2\hat x_2x^2​, and there is no path from x3x_3x3​ that is allowed to reach the output layer. Here is an example of a 1-layer autoencoder that respects the autoregressive structure, the validity of which you can verify through tracing the paths:

In practical implementation where there are many neurons and layers, we first label the input and output neurons with the given ordering. Then, for each unit in the hidden layer, we label it by a random integer i∈{1,…,n−1}i \in \{1,\dots, n-1\}i∈{1,…,n−1}. Any unit with label iii in the hidden layers can only connect to neurons in previous layers that are labeled {1,…,i}\{1,\dots, i\}{1,…,i}. Finally, any unit in the output layer with label iii can only connect to neurons in the previous layer that are labeled {1,…,i−1}\{1,\dots, i-1\}{1,…,i−1}. These rules immediately enforce that x^i\hat x_ix^i​ can only depend on x<i\mathbf{x}_{<i}x<i​. Finally, let’s see how we can use masking matrices to convert a fully connected autoencoder into the valid form as above: To make the masking matrices as illustrated above, we simply label the corresponding columns and rows with the unit labels. For the hidden layers, we mask all cells that have column label greater than row label. For the output layer, we mask all cells that have column label greater than or equal to row label.

Final Thoughts on Autoregressive Models

  1. We have analytical form of p(x)=∏i=1np(xi∣x<i)p(\mathbf x) = \prod_{i=1}^n p(x_i | \mathbf{x}_{<i})p(x)=∏i=1n​p(xi​∣x<i​). This means that the log-likelihood can be evaluated easily, especially via using MADE for parallel computing. Thus, training can be done efficiently.
  2. Sequential sampling can be done easily, BUT can be slow because it requires sequential generation.
  3. These models require an ordering, and different orderings can lead to different result (because we assume a compact representation for parameterization).
  4. There is no natural way to get features via unsupervised learning.
